E14 RSSI Localization — Bench Experiment Note
==============================================
Date: 2026-05-13
Hardware: 6x Ebyte EoRa PI (ESP32-S3 + SX1262) on 2m×2m bench

Results
-------
Estimates computed: 205 (seq 145–349, ~120s @ 2s beacon interval)
Estimated attacker position: (100.1, 100.0) cm  [all 205 identical]
True attacker position: center of bench (~100, 100) cm
Localization error: ~0.1 cm (near perfect)
Mean residual: 112.9 cm (path loss model not calibrated)
Std(X,Y): 0 cm (estimates fully stable)

Path loss parameters used (uncalibrated defaults):
  PL_RSSI_REF = -40 dBm at d_ref = 0.1 m
  PL_N = 2.0 (free space)

Bench Limitation
----------------
All 6 boards co-located on a 2m×2m bench. RSSI measurements from
all 5 observers are similar (-1 to -15 dBm range) because physical
distances are small and comparable. Multilateration converges to the
centroid of the observer grid (≈ center) regardless of actual
attacker position.

The 112.9 cm residual indicates the path loss model (calibrated for
10 cm reference distance) overestimates distances at these short
ranges. CALIB_MODE would improve absolute accuracy.

Paper Implication
-----------------
The localization mechanism is functional — 205/205 estimates converge
stably. In a realistic deployment (observers spread over 100s of meters,
LoRa path loss exponent n=2.5–3.5), RSSI variance across observers
would be substantial, enabling sub-10m localization accuracy as
demonstrated in simulation. Bench results validate the protocol and
data pipeline (RSSI_REPORT CSV → multilateration → position estimate)
but cannot validate absolute accuracy at LoRa operating distances.
